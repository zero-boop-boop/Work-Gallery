# Codex (APP) 保姆级全攻略

> 一期精通 Codex 全部功能，涵盖 12 个章节，从安装到高阶自动化。

---

## 一、安装准备

- **前置依赖**：需提前安装 Git、Node.js、VS Code
- **下载安装**：访问 Codex 官网，官网会根据操作系统（Windows / Mac）自动提供对应安装包
- **登录方式**：支持 ChatGPT 账户登录，**免费账户也可使用**（额度较低）
- **初始化设置**：
  - 选择希望 Codex 处理的工作类型，会预装对应插件和 Skills
  - 选择主要使用场景（编程 or 日常工作），后续可在设置中修改
  - 点击"设置沙盒"按钮完成沙盒初始化

> **Windows 与 Mac 功能基本一致**，唯一差异：Mac 独有 **Computer Use**（自动操作电脑）功能。

## 二、界面布局

Codex App 采用经典的 **三栏布局**：

| 区域 | 内容 |
|------|------|
| 左侧 | 任务列表（对话 + 项目） |
| 中间 | 对话窗口（权限控制、上下文、模型选择） |
| 右侧 | 多功能区域（浏览器、文件树等） |

### 任务管理

- **新建对话**：`Ctrl+N`（Mac: `Cmd+N`），可选择归属项目
- **搜索历史**：`Ctrl+G`（Mac: `Cmd+G`），仅支持搜索对话标题
- **对话标题**：由 AI 根据对话内容自动生成，可双击重命名
- **归档对话**：点击归档按钮，可在"设置 → 已归档对话"中恢复
- **无项目对话**：统一收纳在"对话"栏底部

## 三、权限与沙箱系统

Codex 的权限控制 **完全围绕沙箱展开**，与 Claude Code 有本质区别：

- **Claude Code**：沙箱是额外的保护层，需手动开启
- **Codex**：沙箱是整个权限系统的 **地基**，项目文件夹即沙箱

### 沙箱的两个硬性限制

1. **不能修改沙箱外的文件**
2. **沙箱禁止联网**

> 这两个限制 **不是靠模型自觉遵守**，而是使用操作系统底层机制实现的（如 macOS 使用 Seatbelt Sandbox）。

### 权限三档模式

| 档位 | 行为 | 推荐度 |
|------|------|--------|
| **一档：人工审查** | 所有提权操作（Escalate）需人工批准 | 安全但繁琐 |
| **二档：自动审查**（推荐） | AI 小模型自动判断风险，低风险放行，高风险才需人工 | 安全 + 便利 |
| **三档：完全访问** | 完全无视沙箱限制，可执行一切操作（有醒目风险提示） | 仅特殊场景 |

**默认权限**：Codex 可直接修改沙箱内所有文件，无需逐个审批。

## 四、上下文管理

- **上下文使用量**：对话窗口右侧圆圈显示当前对话历史占用的模型上下文比例
- **自动压缩**：上下文超限时，Codex 自动压缩历史对话
- **手动压缩**：输入 `/` 选择压缩选项
- **最佳实践**：**清空好于压缩** — 完成一个任务后开启新对话，避免历史对话干扰 AI 注意力

### 模型与速度

- **模型选择**：选择最新模型（如 ChatGPT 5.5）
- **速度选项**：
  - **标准**：正常速度，正常消耗
  - **快速**：推理速度提升 50%，但消耗 **双倍套餐用量**
- **套餐额度**：左下角"设置 → 剩余额度"查看，有 **5 小时限额** 和 **周限额**，任一触顶即停用

## 五、核心功能详解

### 5.1 AI 绘图

- 内置 **GPT Image 2** 生图模型
- 在对话中可直接要求 AI 生成/替换图片
- 支持通过截图粘贴（`Ctrl+V` / `Cmd+V`）提供视觉参考

### 5.2 Steer（引导/接管）

**Steer** = 中途接管方向盘，纠正 AI 执行方向。

- **场景**：发现 AI 理解有误或执行方向不对时使用
- **操作**：点击"引导"按钮（或 `Ctrl+Enter`），立即中断当前执行并注入新指令
- **对比**：默认模式下新指令进入队列排队，需等上一轮执行完

> **设置路径**：设置 → 常规设置 → 跟进行为。推荐默认选择"排队"，需要时手动触发引导。

### 5.3 计划模式（Plan Mode）

- 开启后 Codex **先输出完整工作计划**，确认后再执行
- 倾向使用 **问题卡片** 与用户对齐细节
- **适用场景**：所有复杂任务，确保 AI 精准理解意图

示例流程：开启计划 → 输入需求 → AI 生成计划 → 用户确认 → AI 执行 → 内置浏览器自动测试

### 5.4 批注功能

- 在内置浏览器中点击"批注"按钮 → 选中页面元素 → 添加评论
- AI 会根据批注自动修改代码

## 六、Git 与 GitHub 集成

### 基础操作

- 要求 AI 初始化 Git 仓库：自动生成 `.gitignore` 并完成初始化
- 初始化后可通过右上角按钮用 VS Code 等 IDE 打开项目
- 支持对话式 Git 操作：提交、推送到 GitHub 等

### 进阶技巧 1：对话 + 代码回滚

1. **对话回滚**：使用 **Fork（分叉）** 功能，在某对话位置点击分叉 → "派生到本地"，新对话自动剔除后续内容
2. **代码回滚**：在 VS Code Source Control 中找到目标 Commit → 复制 Commit Hash → 让 AI 执行 `git reset` 回退

> Fork 只能回退对话历史，不能同步回退代码，需两者配合操作。

### 进阶技巧 2：Git Worktree

- **本质**：基于主干创建新分支，复制到独立文件夹
- **优势**：多个分支在不同文件夹并行开发，互不干扰，随时可合并回主干
- **操作**：项目右键 → 创建永久工作树 → 命名 → 在各分支中并行开发 → 合并回主干 → 移除临时分支

## 七、云端运行环境

- **前提**：代码需先同步到 GitHub
- **操作**：运行模式选择"关联 Codex Web" → 右上角"前往 Cloud" → 选择仓库 → 输入需求
- **流程**：AI 初始化云端环境 → 拉取 GitHub 代码 → 分析并修改 → 创建 Pull Request → 用户审核合并
- **本地同步**：VS Code Source Control 中点击"SYNC Changes"拉取最新代码

> **移动端场景**：Codex 网页版可用手机打开，出门在外时让云端执行任务，手机上审批即可。

## 八、记忆系统

每次新建对话，AI 对项目记忆为 **空白**。两种方式为 Codex 增加记忆：

### 8.1 项目级 agents.md

- **位置**：项目根目录下的 `agents.md`
- **作用**：AI 每次对话自动读取，作为上下文带入
- **内容**：项目背景、技术栈、个人偏好等
- **可让 AI 自动编写**：提示"通读当前项目，把学到的知识保存到 agents.md"
- **通用性**：Claude Code、Open Code 等工具也通用

### 8.2 全局 agents.md

- **位置**：设置 → 个性化 → 自定义指令
- **作用**：对本机所有项目生效
- **实际文件**：`C:\Users\<用户名>\.codex\agents.md`

**推荐全局安全规则**：
```
禁止使用脚本批量删除文件或目录，只能使用 Remove-Item 逐个删除。
如需批量删除，应停止操作让用户手动处理。
```

### 8.3 实验性记忆功能

- 设置中"启用记忆"：可从聊天中自动生成记忆并带入新聊天（实验阶段）

## 九、插件与 Skills

### 插件（第三方服务集成）

- **入口**：左侧边栏 → 插件
- 安装后通过 `/` 调用

**实用插件举例**：

| 插件 | 功能 |
|------|------|
| GitHub | 查询仓库信息、Star 数等 |
| Gmail | 读写邮件、自动生成摘要 |
| Netlify | 部署静态网页 / Next.js 项目（国内可直连） |
| Computer Use | 自动操作电脑（仅 Mac） |

### 自动化

- 将对话中的工作流编制为定时任务
- **入口**：左侧边栏 → 自动化
- 可设置重复频率（每周 / 每天 / 每小时）、使用模型
- AI 执行中积累的经验会写入 `memory.md`，辅助后续执行
- **Mac 特有**：设置中开启"运行时防止系统休眠"确保定时任务正常执行

### Skills（专业技能包）

Skills 是给 AI Agent 的 **带目录的说明书**，封装工作流和专业能力。三种安装方式：

1. **官方 Skills**：插件市场中的技能标签页
2. **第三方 Skills**：从 GitHub 下载 → 放入项目 `.codex/skills/` 目录
3. **自建 Skills**：使用内置 `skill creator` 技能，描述工作流让 AI 自动生成

**自建 Skill 示例**：视频字幕 → Markdown 图文笔记
- 提供视频 + 字幕文件 → AI 读取字幕 → 生成 Markdown → 调用 ffmpeg 在关键位置截图 → 替换截图占位符

## 十、MCP（模型上下文协议）

MCP = AI 大模型的 **标准化工具箱**，让模型与外界互动。

### 配置步骤

1. **入口**：左下角设置 → MCP 服务器 → 添加服务器
2. **以 Supabase 为例**：
   - Supabase 个人主页 → 创建项目 → Connect → MCP → 客户端选 Codex → 复制 URL
   - Codex 中选择 HTTP 传输方式，填入 URL，命名
   - 终端执行 `codex mcp login supabase` → 浏览器授权
   - **授权后需重启 Codex**

### 实战：添加后端数据库

- 让 AI 调用 Supabase MCP 创建数据表
- AI 自动在后端添加端点，使用 Postgres 连接池写入数据
- 修改 `.env.local` 填入连接地址 → 重启 → 测试表单提交

## 十二、Computer Use（Mac 独有）

让 Codex 直接操作电脑完成任务。

- **前提**：安装 Computer Use 插件（仅 Mac）
- **调用方式**：输入 `@` → 选择 Computer Use → 描述任务
- **特点**：使用虚拟鼠标在后台运行，不占用当前窗口

**应用场景**：
- 自动打开聊天软件发送消息
- 浏览器打开 GitHub 看板 → 汇总项目进度 → 生成英文简报 → 发送给老板
- 结合自动化功能，每天定时执行
